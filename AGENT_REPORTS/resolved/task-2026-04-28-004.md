# Task 004 — Hide PRO badges in LayerPanel for authenticated Pro users (P1)

Date: 2026-04-28

## Context

The pro P1 test was designed to verify that Pro users see zero PRO badges in the
LayerPanel (no gating chrome needed when the user already has access). When the
pro suite ran in earlier sessions the annotation showed `pro-badge-count: 8` against
an expected value of 0. This is a confirmed bug, not a phantom.

Root cause is a single line in `LayerRow` in `src/components/LayerPanel.jsx`:

```jsx
// Line 114 — current (wrong):
{layer.pro && <ProBadge />}

// Should be:
{layer.pro && !isPro && <ProBadge />}
```

`isPro` is already a prop on `LayerRow` (line 86) and the `disabled` flag on
line 87 already uses it correctly (`const disabled = layer.pro && !isPro`).
The badge render simply forgot the `!isPro` guard.

This means every Pro subscriber currently sees PRO badges on layers they have
full access to — the same visual treatment as a free user, which is confusing
and makes the UI look broken for paying customers.

`LayerPanel.jsx` is NOT in the shared-file list (AGENTS.md) — no INTENT block
required.

## Task

In `src/components/LayerPanel.jsx`, line 114, change:

```jsx
{layer.pro && <ProBadge />}
```

to:

```jsx
{layer.pro && !isPro && <ProBadge />}
```

That is the entire fix. Do not touch anything else in this file.
