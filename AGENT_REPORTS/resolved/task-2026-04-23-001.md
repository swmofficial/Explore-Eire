# Task 001 — Fix BottomNav: render `<nav>` element so Playwright test passes

## Context

The Playwright test `bottom nav is visible` uses `page.locator('nav').first()` and calls
`expect(nav).toBeVisible()`. It fails because `BottomNav.jsx` renders a `<div>` as its
root element — not a `<nav>`. There is no CSS hiding the element; the component is mounted
correctly in `App.jsx` (line 85, inside the fixed wrapper, after the splash screen). The
test waits 2500 ms before asserting, which is enough for the 1800 ms + 300 ms splash.

The fix is a one-line change: swap the root `<div>` to `<nav>` in `BottomNav.jsx`. All
existing inline styles stay identical. No other file needs touching.

## Task

In `src/components/BottomNav.jsx`, change the root element of the `return` statement in
the default export (line 83) from `<div` / `</div>` to `<nav` / `</nav>`.

Exact change — open tag (line 83):

```jsx
// BEFORE
    <div
      style={{
        position: 'fixed',
        ...
      }}
    >

// AFTER
    <nav
      style={{
        position: 'fixed',
        ...
      }}
    >
```

And the matching closing tag near the end of the component:

```jsx
// BEFORE
    </div>

// AFTER
    </nav>
```

No other changes. Do not add `role`, `aria-label`, or any other attributes. Do not touch
App.jsx.
