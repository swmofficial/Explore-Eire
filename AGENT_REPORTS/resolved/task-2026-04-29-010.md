# Task 010 — Playwright geolocation permission + mock location

Date: 2026-04-29
Status: DONE
Source: ux-findings-2026-04-29.md (findings 1, 2, 6)
Branch: main

## Context

Three findings share one root cause: Playwright tests run with NO geolocation permission
and no mock GPS position. The affected flows are:

- **P3 (pro)** — `pro P3` fails with `expect(locator).not.toBeDisabled()` on the Save
  Waypoint button. `WaypointSheet.AddMode` calls `getCurrentPosition()` on mount. In the
  test environment this call rejects (no permission → no mock position), `coords` is never
  set, `canSave = !!coords && !saving = false`, button stays disabled.
- **V1 (pro)** — `pro V1` annotation: `ee_session_trail empty or missing (V1 confirmed)`.
  GPS tracking calls `appendSessionTrailPoint` for each GPS fix. Without geolocation, no
  fixes are delivered, no points accumulate, `ee_session_trail` key is never written.
  App code (mapStore.js:59-73, task-006) is CORRECT. This is a PHANTOM finding from the
  test infrastructure.
- **V11 (guest)** — `guest V11` annotation: `ee_guest_waypoints absent after reload`. The
  test taps the camera button, but if WaypointSheet requires GPS to allow Add mode, no
  waypoint is added, `addSessionWaypoint` is never called, and the key is never written.
  App code (mapStore.js:77-91, task-002) is CORRECT. This is a PHANTOM finding.

`playwright.config.js` currently has no `geolocation` or `permissions` setting in the
`use:` block.

## Task

Edit `playwright.config.js` — add two fields to the existing `use:` block:

```js
use: {
  baseURL: process.env.BASE_URL || 'http://localhost:5173',
  screenshot: 'only-on-failure',
  video: 'retain-on-failure',
  viewport: { width: 390, height: 844 },
  userAgent: '...',          // unchanged
  geolocation: { latitude: 53.3498, longitude: -6.2603 },  // Dublin city centre
  permissions: ['geolocation'],
},
```

No other files need changing. The `geolocation` field provides a mock position that
`getCurrentPosition()` will resolve with. The `permissions` grant removes the browser
permission prompt so the mock is used immediately.

## Constraints

- `playwright.config.js` is not a shared app source file — no INTENT block required.
- Do NOT change any application source files (WaypointSheet.jsx, mapStore.js, etc.).
- The coordinates must be a valid location in Ireland (the GPS mock will be logged).
  Dublin (53.3498, -6.2603) is appropriate.
- This change applies to ALL three suites (guest, free, pro) — no per-spec change needed.

## Definition of Done

- `playwright.config.js` has `geolocation` and `permissions` in the top-level `use:`.
- `pro P3` should now reach the save button in an enabled state after the name field is
  filled. The `expect(save).not.toBeDisabled()` assertion passes.
- `pro V1` annotation should change: if GPS tracking ran and points accumulated, the
  `ee_session_trail` key will be present. The "V1 confirmed" annotation changes to
  "ee_session_trail present: [data]".
- `guest V11` annotation should change: after camera tap + dismiss, if `addSessionWaypoint`
  was called, `ee_guest_waypoints` will be present after reload.
