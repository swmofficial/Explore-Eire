# Task 005 — Fix V10: useAuth must not overwrite persisted isPro on offline auth loss
Date: 2026-04-28
Status: PENDING
Source: ux-findings-2026-04-28.md (Finding 1 — V10)
Branch: main

## Context

The persist middleware added in task-001 (commit d84b479) correctly writes
`isPro` and `subscriptionStatus` to localStorage under the key `ee-user-prefs`.
On a clean reload that key hydrates before any React code runs.

However, `useAuth.js` (`src/hooks/useAuth.js`) contains this block in its
`onAuthStateChange` listener:

```js
} else {
  setUser(null)
  setIsPro(false)
  setSubscriptionStatus('free')
  legalFetchedFor.current = null
}
```

This fires whenever Supabase delivers a null session. Null sessions arrive in
two distinct cases:

1. **Genuine sign-out** (`SIGNED_OUT` event, user-initiated): `isPro` MUST be
   reset to false — the user is no longer authenticated.

2. **Offline JWT refresh failure** (`SIGNED_OUT` or no session, network down):
   The JWT in localStorage has expired and Supabase cannot issue a new one
   while offline. The auth library fires a null session even though the user
   did not sign out. Resetting `isPro` here discards the correctly-persisted
   value and locks a paying Pro user out of features they've paid for.

The UX Agent re-confirmed V10 ("Pro Subscription Status Lost on Offline
Reload") in the 2026-04-28 run because the test exercises exactly case 2. The
persisted value is written but immediately overwritten by useAuth.

The `onAuthStateChange` callback receives an `event` string as its first
parameter. Using it to distinguish the two cases is the correct fix.

## Task

Modify `src/hooks/useAuth.js` — the `onAuthStateChange` callback — so that
`isPro` and `subscriptionStatus` are only reset when the event is a deliberate
sign-out or when the device is online.

**Exact change** (no file is modified except `useAuth.js`):

```js
// BEFORE:
const { data: { subscription } } = supabase.auth.onAuthStateChange((_event, session) => {
  if (session?.user) {
    handleSignedIn(session.user)
  } else {
    setUser(null)
    setIsPro(false)
    setSubscriptionStatus('free')
    legalFetchedFor.current = null
  }
})

// AFTER:
const { data: { subscription } } = supabase.auth.onAuthStateChange((event, session) => {
  if (session?.user) {
    handleSignedIn(session.user)
  } else {
    setUser(null)
    legalFetchedFor.current = null
    // Only clear subscription state on genuine sign-out or when online.
    // If offline, the session was lost due to JWT expiry — persisted Pro
    // status is the best data available; do not overwrite it.
    if (event === 'SIGNED_OUT' || navigator.onLine) {
      setIsPro(false)
      setSubscriptionStatus('free')
    }
  }
})
```

Also rename the `_event` parameter to `event` on the first character so the
linter doesn't complain about the used variable.

**`useAuth.js` is NOT on the shared-files list** (see AGENTS.md). No INTENT
block is required before modifying it.

## Constraints

- Do not touch any other file.
- Do not introduce a separate offline-detection hook or global variable.
- `navigator.onLine` is the correct check: it is synchronous, always available
  in the browser, and returns false in genuine offline scenarios.
- If `navigator.onLine` is unavailable (Capacitor native edge case), the
  `onLine` property defaults to `true` in most environments, so the
  reset will still fire — an acceptable safe-default.
- Preserve the existing `useEffect` dependency comment (`// eslint-disable-line`).

## Definition of Done

- `src/hooks/useAuth.js` updated exactly as specified.
- A Pro user who reloads the app while offline retains their `isPro: true`
  state (the persisted localStorage value is no longer overwritten).
- A genuine sign-out (online) still correctly resets `isPro` to false.
- No other files changed.
