## Context

The tier-1 Playwright pipeline (`ux-test.yml`) runs "smart targeting" via
`.github/scripts/get-test-targets.js`. It maps changed source files to spec names and
writes them to `test-targets.txt`. The ux-test job then runs:

```
TARGETS=$(cat test-targets.txt | tr '\n' ' ')
npx playwright test $TARGETS
```

With `playwright.config.js` `testDir: './tests/ux'`, Playwright resolves those spec names
inside `tests/ux/`. The problem: `get-test-targets.js` maps to spec names that do not
exist:

| Spec name referenced | Exists in tests/ux/? |
|---|---|
| `app.spec.js`        | Ôťů yes |
| `auth.spec.js`       | Ôťů yes |
| `mobile.spec.js`     | ÔŁî missing |
| `map.spec.js`        | ÔŁî missing |
| `sheets.spec.js`     | ÔŁî missing |
| `dashboard.spec.js`  | ÔŁî missing |
| `module.spec.js`     | ÔŁî missing |
| `settings.spec.js`   | ÔŁî missing |
| `learn.spec.js`      | ÔŁî missing |
| `paywall.spec.js`    | ÔŁî missing |

When Playwright is invoked with missing file arguments it errors on them but `|| true`
swallows the failure. Only the specs that DO exist (`app.spec.js`, sometimes `auth.spec.js`)
produce results. This is why Gemini's tier-1 reports only ever show `app.spec.js` outcomes,
even when map or BottomNav files were changed ÔÇö the smart-targeting logic fires correctly
but the target specs don't exist.

Note: the tier-2 audit pipeline (`ux-audit.yml`) is separately gated ÔÇö it only triggers
after Ôëą3 consecutive failures (streak threshold in `update-streak.js`). That is intentional
and does NOT need changing. The audit specs in `tests/audit/` are unaffected by this task.
