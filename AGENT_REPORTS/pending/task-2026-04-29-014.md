# Task 014 — Fix V8/V9: switch basemap + layerVisibility to manual IIFE persistence

Date: 2026-04-29
Status: PENDING
Source: ux-findings-2026-04-29.md + Architect analysis
Branch: main

## Context

The UX Agent report for 2026-04-29 shows:
- `guest V9` FAIL: `Test timeout of 60000ms exceeded.` (basemap not persisted across reload)
- `free V8` FAIL: `Test timeout of 60000ms exceeded.` (layer visibility not persisted across reload)

Both V8 and V9 are being caused by the same root cause as V7 (theme) and V15 (activeModule):
**Zustand `persist` middleware is unreliable in the deployed Vercel/Capacitor environment.**

The established fix pattern — proven by task-008 (V7) and task-013 (V15) — is to:
1. Remove the field from Zustand `persist` partialize
2. Read from localStorage via an IIFE in the state initialiser
3. Write to localStorage explicitly in the setter

`basemap` and `layerVisibility` are currently persisted via `ee-map-prefs` (Zustand persist).
The persist key has NOT been confirmed to work in the deployed environment.
Given that the identical pattern failed for `ee-user-prefs` (theme) and `ee-module-prefs` (activeModule),
it is very likely `ee-map-prefs` is also silently failing to write on the deployed instance.

### Source files

- `src/store/mapStore.js` — the store containing `basemap` and `layerVisibility`
- `tests/ux/guest.spec.js` — V9 test (basemap)
- `tests/ux/free.spec.js` — V8 test (layer visibility)

### Current state (mapStore.js)

`basemap` and `layerVisibility` are in the `partialize` of the Zustand persist config
(key: `ee-map-prefs`). There is no manual IIFE read or explicit setItem in the setters.

### Proven working pattern (from task-008, task-013)

```js
// IIFE init — reads from localStorage on store creation:
basemap: (() => {
  try { return localStorage.getItem('ee_basemap') || 'satellite' } catch { return 'satellite' }
})(),

// Explicit write in the setter:
setBasemap: (basemap) => {
  try { localStorage.setItem('ee_basemap', basemap) } catch {}
  set({ basemap })
},
```

For `layerVisibility` (an object, not a string):
```js
layerVisibility: (() => {
  try {
    return JSON.parse(localStorage.getItem('ee_layer_visibility') || 'null') || { stream_sediment: true }
  } catch { return { stream_sediment: true } }
})(),

setLayerVisibility: (layerId, visible) =>
  set((state) => {
    const next = { ...state.layerVisibility, [layerId]: visible }
    try { localStorage.setItem('ee_layer_visibility', JSON.stringify(next)) } catch {}
    return { layerVisibility: next }
  }),
```
