## Context

We have designed and built a specialist UX Agent system that replaces the generic
Gemini analysis pipeline. The current system has an ~80% phantom rate — Gemini reads
test output, guesses what might be wrong, and produces findings that don't exist in
the app. The new system fixes this with three mechanisms:

1. **STATE_MAP.md** — architectural ground truth the agent reasons against (what persists,
   what dies on unmount, what fails offline, where domains touch)
2. **Structured prompt** — forces confidence scoring, evidence requirements, and explicit
   "cannot confirm" fields so the agent must be honest about what it can't see
3. **Feedback loop** — resolved tasks with CONFIRMED/PHANTOM/MISDIAGNOSED verdicts are
   fed back into the agent as calibration data, teaching it which patterns are real

This task adds all new files and modifies two existing files (AGENTS.md, plus pending/
tasks need resolution verdicts and moving to resolved/). No application source code
is touched. No shared stores, components, or styles are modified.

### Part 2 — Create UX Agent pipeline script

**File: `.github/scripts/ux-agent.js`**

This script replaces `gemini-analysis.js` as the primary analysis engine. It:

- Loads `BRAIN/STATE_MAP.md` and `BRAIN/agents/ux-agent-context.md`
- Loads `playwright-report.json` and git diff/log
- Walks `test-results/` directory for PNG/JPG screenshots, encodes as base64
- Walks `AGENT_REPORTS/resolved/` for task files, parses Verdict from each, builds
  accuracy stats (confirmed%, phantom%, misdiagnosed%) and recent verdicts list
- Injects all of the above into a structured system prompt for Gemini
- Calls `gemini-2.5-flash` with `temperature: 0.2` and max 4096 output tokens
- Writes findings to `AGENT_REPORTS/ux-findings/ux-findings-YYYY-MM-DD.md`
- Also writes to `AGENT_REPORTS/report-YYYY-MM-DD.md` for backward compatibility

Copy verbatim from companion zip. ~12KB.
