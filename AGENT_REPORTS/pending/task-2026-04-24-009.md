## Context

Three surface-level spec files are effectively no-ops:

| File | Current state |
|---|---|
| `tests/ux/settings.spec.js` | goto('/'), wait 3s, screenshot, assert zero errors |
| `tests/ux/dashboard.spec.js` | goto('/'), wait 3s, screenshot, assert zero errors |
| `tests/ux/learn.spec.js` | goto('/'), wait 3s, screenshot, assert zero errors |

None of them actually navigate to their respective surfaces before asserting. Because
`App.jsx` starts on `activeTab = 'dashboard'`, all three specs are testing the DashboardView —
not Settings, not Learn. A complete crash of `SettingsView` or `LearnView` would pass them all.

Additionally, `dashboard.spec.js` has a test called `'dashboard tab visible in nav'` that
takes a screenshot but doesn't assert anything visible in the nav or the dashboard surface
itself. It passes because it has no assertions.

**What the code tells us (App.jsx):**
```js
{activeTab === 'dashboard' && <DashboardView onNavigate={setActiveTab} />}
{activeTab === 'settings' && <SettingsView onNavigate={setActiveTab} />}
{activeTab === 'learn' && <LearnView />}
{activeTab === 'profile' && <ProfileView onNavigate={setActiveTab} />}
```

The map tab hides/shows via CSS visibility (always mounted). All other tabs conditionally
render their surface component. If a component crashes on mount, console errors fire.

**SettingsView detail:** `SettingsPanel.jsx` has `aria-label="Close settings"` on a close
button — we can use `page.getByLabel('Close settings')` as a content assertion for Settings.

**INTENT analysis:** None of these files are shared files per AGENTS.md. No INTENT block needed.

## Task

Rewrite all three spec files. Each should:
1. Click the relevant tab button via `getByRole('button', { name: '...' })`
2. Wait 1500ms for the surface to render
3. Take a screenshot
4. Assert a visible element specific to that surface (proving the component mounted)
5. Assert zero new console errors from the navigation

### 1. Rewrite `tests/ux/settings.spec.js`

```js
import { test, expect } from '@playwright/test';

test.beforeEach(async ({ page }) => {
  page._errors = [];
  page.on('console', m => { if (m.type() === 'error') page._errors.push(m.text()); });
  page.on('pageerror', e => page._errors.push(e.message));
  await page.goto('/');
  await page.waitForTimeout(2500);
  await page.getByRole('button', { name: 'Settings' }).click();
  await page.waitForTimeout(1500);
});

test('settings surface renders', async ({ page }) => {
  await page.screenshot({ path: 'tests/ux/screenshots/settings-01.png' });
  // SettingsView renders — nav should still be present
  const nav = page.locator('nav').first();
  await expect(nav).toBeVisible();
});

test('no settings errors on navigation', async ({ page }) => {
  expect(page._errors).toHaveLength(0);
});
```
