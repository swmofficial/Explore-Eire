# Task 013 — V15 activeModule resets: switch moduleStore to manual localStorage

Date: 2026-04-29
Status: PENDING
Source: ux-findings-2026-04-29.md (finding 7)
Branch: main

## Context

`guest V15` annotation: `ee-module-prefs absent after reload (V15 confirmed)`. task-001
(d84b479) wrapped `moduleStore` with Zustand's `persist` middleware, partializing
`activeModule` to `ee-module-prefs`. The test (updated in task-009) checks
`localStorage.getItem('ee-module-prefs')` directly after reload and confirms the key is
absent.

This is the same class of failure as the `basemap`/`layerVisibility` persist issues in
`mapStore` and the original `theme` failure in `userStore` (fixed in task-008 by switching
to manual IIFE pattern). Zustand v5's persist middleware is not reliably writing to
`localStorage` in the deployed environment for these stores.

The proven-reliable pattern in this codebase is:
- **Read**: IIFE at store creation reads from a dedicated localStorage key.
- **Write**: Action writes directly to localStorage on every call.
- Confirmed working: `ee_onboarded` (bypassOnboarding), `ee_theme` (task-008),
  `ee_session_trail` (task-006), `ee_guest_waypoints` (task-002).

## Task

### Part A — src/store/moduleStore.js (INTENT REQUIRED — see below)

Remove the `persist` wrapper and switch `activeModule` to the manual IIFE pattern.
Use key `ee_active_module` (simple string, no JSON nesting).

**Before:**
```js
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

const useModuleStore = create(
  persist(
    (set) => ({
      activeModule: 'prospecting',
      setActiveModule: (moduleId) => set({ activeModule: moduleId }),
      ...
    }),
    {
      name: 'ee-module-prefs',
      partialize: (state) => ({ activeModule: state.activeModule }),
    },
  ),
)
```

**After:**
```js
import { create } from 'zustand'

const useModuleStore = create((set) => ({
  activeModule: (() => {
    try { return localStorage.getItem('ee_active_module') || 'prospecting' } catch { return 'prospecting' }
  })(),
  setActiveModule: (moduleId) => {
    try { localStorage.setItem('ee_active_module', moduleId) } catch {}
    set({ activeModule: moduleId })
  },

  accessibleModules: [],
  setAccessibleModules: (modules) => set({ accessibleModules: modules }),

  activeCategoryTab: null,
  setActiveCategoryTab: (tab) => set({ activeCategoryTab: tab }),

  activeSurface: 'map',
  setActiveSurface: (surface) => set({ activeSurface: surface }),
}))
```

Remove the `import { persist }` line entirely (it will no longer be used).
