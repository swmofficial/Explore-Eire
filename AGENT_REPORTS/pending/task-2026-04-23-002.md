# Task 002 — Replace CSS variables in MapLibre paint properties with static hex values
Date: 2026-04-23
Status: DONE
Source: report-2026-04-23.md
Branch: agent/implementer

## Context

MapLibre GL JS v5.22 does not support CSS custom properties (`var(--...)`) in layer `paint`
objects. When it encounters them it logs `color expected, "var(--color-accent)" found` and
falls back to a default (often transparent or black), breaking the visual appearance of
route lines, waypoint dots, session trail, mineral gold layer, and mineral quartz layer.

The affected file is `src/components/Map.jsx`. The errors come from `addDataLayers()` which
defines all MapLibre sources/layers. Two CSS variables are used:

- `var(--color-accent)` — resolves to `#E8C96A` (Eire Gold 400, the PRIMARY ACCENT defined
  in ARCHITECTURE.md Design System › Colour Palette)
- `var(--color-text)` — resolves to `#E8EAF0` (Primary text on dark bg, same reference)

**Architecture decision (human-reviewed per report):** Use static hex values (Option A).
Theme-driven dynamic MapLibre colours are a post-launch concern. The map canvas is always
dark-themed regardless of the UI theme (ARCHITECTURE.md Design Rule #2), so hardcoding the
dark-theme values is correct and permanent, not a hack.

## Task

In `src/components/Map.jsx`, replace every occurrence of `'var(--color-accent)'` with
`'#E8C96A'` and every occurrence of `'var(--color-text)'` with `'#E8EAF0'` **only where
they appear inside MapLibre `paint` property objects** (passed to `map.addLayer()`).

Exact locations (all in `addDataLayers()` function):

1. **Line 162** — `MINERAL_LAYERS` array, `mineral-gold` entry:
   `color: 'var(--color-accent)'` → `color: '#E8C96A'`

2. **Line 166** — `MINERAL_LAYERS` array, `mineral-quartz` entry:
   `color: 'var(--color-text)'` → `color: '#E8EAF0'`

3. **Line 409** — `session-trail-line` layer paint:
   `'line-color': 'var(--color-accent)'` → `'line-color': '#E8C96A'`

4. **Line 432** — `session-dots` layer paint:
   `'circle-color': 'var(--color-accent)'` → `'circle-color': '#E8C96A'`

5. **Line 454** — `session-waypoints` layer paint:
   `'circle-color': 'var(--color-accent)'` → `'circle-color': '#E8C96A'`

6. **Line 479** — `saved-waypoints-circles` layer paint:
   `'circle-color': 'var(--color-accent)'` → `'circle-color': '#E8C96A'`

7. **Line 500** — `route-builder-line` layer paint:
   `'line-color': 'var(--color-accent)'` → `'line-color': '#E8C96A'`

8. **Line 521** — `route-builder-dots` layer paint:
   `'circle-color': 'var(--color-accent)'` → `'circle-color': '#E8C96A'`

Do NOT touch:
- Lines 1055–1073: these are React JSX inline styles and SVG attributes — CSS variables

- Any other file.
- MINERAL_LAYERS entries that already use literal hex (copper, lead, uranium, silver,
  marble, fluorspar — lines 163–169 minus 162 and 166).
