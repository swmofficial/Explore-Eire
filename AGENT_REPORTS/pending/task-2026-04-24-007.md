## Context

Both recent Gemini reports flag the same root problem: every spec file in `tests/ux/` is a
placeholder. They navigate to `/`, wait 3 seconds, take a screenshot, and assert zero console
errors. No test actually taps a BottomNav tab or verifies that the correct surface renders.

`app.spec.js` currently tests:
- App title
- Splash screen transition (screenshot only)
- Nav visible (screenshot only)
- Map container present (screenshot only)

None of these exercise tab routing. A complete crash of DashboardView, LearnView, SettingsView,
or ProfileView on navigation would pass all current tests undetected.

**What the code tells us (App.jsx):**
- App starts on `activeTab = 'dashboard'` — DashboardView renders by default
- Map is always mounted but hidden (CSS `visibility`) when not on map tab
- Tab switch sets `activeTab`, which switches the rendered surface
- BottomNav buttons have `aria-label` on all five tabs:
  - `aria-label="Settings"`, `aria-label="Dashboard"`, `aria-label="Map"`,
    `aria-label="Learn"`, `aria-label="Profile"`

**INTENT analysis:** `app.spec.js` is not a shared file — no INTENT block required.

## Task

Add five new tests to `tests/ux/app.spec.js` — one per tab — that click the tab, wait for
the surface to render, take a screenshot, and assert no new console errors from the navigation.

Append these tests **after the four existing tests** in `app.spec.js`. Do not remove or
rewrite the existing tests.

```js
test('dashboard tab navigates without errors', async ({ page }) => {
  const errorsBefore = page._consoleErrors.length;
  await page.getByRole('button', { name: 'Dashboard' }).click();
  await page.waitForTimeout(1500);
  await page.screenshot({ path: 'tests/ux/screenshots/tab-dashboard.png' });
  expect(page._consoleErrors.length).toBe(errorsBefore);
});

test('map tab navigates without errors', async ({ page }) => {
  const errorsBefore = page._consoleErrors.length;
  await page.getByRole('button', { name: 'Map' }).click();
  await page.waitForTimeout(2500);
  const canvas = page.locator('canvas').first();
  await expect(canvas).toBeVisible();
  await page.screenshot({ path: 'tests/ux/screenshots/tab-map.png' });
  expect(page._consoleErrors.length).toBe(errorsBefore);
});

test('learn tab navigates without errors', async ({ page }) => {
  const errorsBefore = page._consoleErrors.length;

  await page.waitForTimeout(1500);
  await page.screenshot({ path: 'tests/ux/screenshots/tab-learn.png' });
  expect(page._consoleErrors.length).toBe(errorsBefore);
});

test('settings tab navigates without errors', async ({ page }) => {
  const errorsBefore = page._consoleErrors.length;
  await page.getByRole('button', { name: 'Settings' }).click();
  await page.waitForTimeout(1500);
  await page.screenshot({ path: 'tests/ux/screenshots/tab-settings.png' });
  expect(page._consoleErrors.length).toBe(errorsBefore);
});

test('profile tab navigates without errors', async ({ page }) => {
  const errorsBefore = page._consoleErrors.length;

  await page.waitForTimeout(1500);

});
```

**Note on `page._consoleErrors`:** The existing `app.spec.js` `beforeEach` already uses
`page._consoleErrors` (not `page._errors`) — match that naming exactly.

## Constraints

- Only modify `tests/ux/app.spec.js`. No other file.
- Do not remove existing tests — append only.
- Use `page.getByRole('button', { name: '...' })` — matches the `aria-label` attributes on
  BottomNav buttons. Do not use CSS selectors or text content selectors for nav buttons.
- For the map tab test: assert `canvas` is visible (confirms MapLibre rendered, not just
  that the tab switch didn't crash). This validates bug register rule #2 and #25 indirectly.
- The `errorsBefore` snapshot pattern means the test catches errors introduced specifically
  by the tab switch — not pre-existing errors from page load.
- Keep commit atomic, prefix `[impl]`.
