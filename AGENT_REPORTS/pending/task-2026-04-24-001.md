# Task 001 — Fix ux-audit.yml: replace fragile git diff command
Date: 2026-04-24
Status: PENDING
Source: Architect (session handoff review)
Branch: agent/implementer

## Context

The `ux-audit.yml` Tier 2 pipeline contains a broken git diff command in the
"Collect git diff" step. The current command chains `git log | tail | awk | xargs`
to extract a SHA and diff from it. This fails when:

- The repo has fewer than 10 commits (xargs receives empty input)
- The SHA extracted is ambiguous
- The pipe produces trailing whitespace xargs can't handle

The result is an empty or missing `git-diff.txt`, which means Gemini receives no
diff context in its audit prompt, and the fallback `|| git diff HEAD --stat` does
not redirect output to `git-diff.txt` (the `>` redirect is on the wrong side of
the `||`).

This is the sole blocker for the first clean PR from `dev` to `main`.
The Tier 2 audit is triggered on every PR to main — it must be reliable.

## Task

In `.github/workflows/ux-audit.yml`, find the "Collect git diff" step and
replace the `run` block exactly as follows:

**BEFORE:**
```yaml
      - name: Collect git diff
        run: git log --oneline -10 | tail -1 | awk '{print $1}' | xargs -I {} git diff {}..HEAD --stat > git-diff.txt || git diff HEAD --stat > git-diff.txt
```

**AFTER:**
```yaml
      - name: Collect git diff
        run: |
          git diff HEAD~10..HEAD --stat 2>/dev/null > git-diff.txt || \
          git diff HEAD~5..HEAD --stat 2>/dev/null > git-diff.txt || \
          git diff HEAD~1..HEAD --stat > git-diff.txt
```

This tries HEAD~10 first (maximum context), falls back to HEAD~5, then HEAD~1.
The `2>/dev/null` suppresses the error when the repo doesn't have enough commits.
Output is always written to `git-diff.txt` on the successful branch.

No other files need changing. Do not touch `ux-test.yml`, any scripts, or any
other workflow file.

## Constraints

- `.github/workflows/ux-audit.yml` is NOT a shared file per AGENTS.md — no
  INTENT block required.
- The rest of `ux-audit.yml` must remain completely unchanged.
- Keep the commit atomic and prefix `[impl]`.

## Definition of Done

- `.github/workflows/ux-audit.yml` "Collect git diff" step uses the new multi-
  fallback command.
- The old `git log --oneline -10 | tail -1 | awk ...` chain is gone.
- `git-diff.txt` is always written with content on every run, even on repos with
  fewer than 10 commits.
- No other files are modified in this commit.
