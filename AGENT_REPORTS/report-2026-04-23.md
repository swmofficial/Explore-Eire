2.  **MapLibre `paint` Property Color Errors**
    *   **Description:** Across multiple tests, console errors were reported by MapLibre GL JS, stating "color expected, \"var(--color-accent)\" found" or "color expected, \"var(--color-text)\" found" for various layer `paint` properties (e.g., `line-color`, `circle-color`). This indicates MapLibre is receiving CSS variables instead of explicit color values, which it does not directly support for these properties in v5.22.
    *   **Likely Cause from Git Diff:** The `git diff` shows that `[infra] console error capture in tests` was recently added (commit `67d59ab`), which enabled the detection of these errors. The underlying cause is an application code change (likely in `mapStore.js` or a map layer definition) that introduced CSS variables for MapLibre `paint` properties. This is a compatibility issue with MapLibre GL JS v5.22.
    *   **Severity:** **MEDIUM** - While tests pass, these errors indicate a functional issue with map styling. Layers may not display with the intended theme colors, degrading the visual experience and potentially breaking map features visually.
