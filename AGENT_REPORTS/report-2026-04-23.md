## UX Report — 2026-04-23

### Summary
This report identifies potential UX regressions and a critical infrastructure issue following recent commits, despite basic smoke tests passing. Key concerns include potential visual defects in the Map component's layers due to recent CSS variable replacements and subtle styling or interaction issues in the Bottom Navigation. Crucially, the newly implemented comprehensive Playwright audit tests are not executing or reporting their results, creating a significant blind spot for detecting these and other UX regressions.

### Findings

1.  **Potential Map Styling / Layer Rendering Regression**
    *   **Description:** The commit `360a79e` involved changes to `src/components/Map.jsx` to "replace CSS vars in MapLibre paint objects". This change carries a high risk of introducing visual regressions, potentially causing map layers (e.g., basemaps, WMS overlays, Gold tier data, custom feature styles) to render incorrectly, with wrong colors/opacities, or even fail to appear. The basic "map container is present" Playwright test does not verify the actual visual content or styling of the map.
    *   **Likely Cause:** Commit `360a79e` (`[impl] fix BottomNav nav element + replace CSS vars in MapLibre paint objects`), specifically the modifications within `src/components/Map.jsx`.
    *   **Severity:** High. The map is the central and most critical component of the Explore Eire application. Any visual defect significantly impairs usability.

2.  **Potential Bottom Navigation Visual or Interaction Regression**
    *   **Description:** The `BottomNav.jsx` component was modified in commit `360a79e` as part of a "fix for nav element". While the "bottom nav is visible" Playwright test passed, this does not confirm correct visual styling (e.g., padding, alignment, responsiveness across various mobile viewports), proper z-index stacking (`zIndex: 40`), or expected interaction behavior (e.g., haptic feedback, accurate tap targets). These "fixes" can often introduce subtle, unintended regressions.
    *   **Likely Cause:** Commit `360a79e` (`[impl] fix BottomNav nav element + replace CSS vars in MapLibre paint objects`), specifically the changes within `src/components/BottomNav.jsx`.
    *   **Severity:** Medium. Visual glitches or broken interactions in the primary navigation degrade the overall user experience and app quality.

3.  **Incomplete Playwright Audit Results - Comprehensive Tests Not Executed/Reported**
    *   **Description:** Commit `9361c47` introduced a "two-tier Playwright system" with an extensive set of `full-audit` tests (e.g., `full-map.spec.js`, `full-mobile.spec.js`, `full-auth.spec.js`). However, the provided `PLAYWRIGHT RESULTS` only show outcomes for `app.spec.js`, which are basic smoke tests. This means the newly added comprehensive audit tests, crucial for detecting detailed UX/UI regressions, were either not executed as part of this run or their results were not included in the report. This significantly compromises our ability to detect regressions, especially for the high-risk changes identified above.
    *   **Likely Cause:** Commit `9361c47` introducing the new audit structure. Potential misconfiguration in the CI workflow files (`.github/workflows/ux-test.yml`, `.github/workflows/ux-audit.yml`) or `playwright.config.js` that prevents the `tests/audit/` suite from running or being reported.
    *   **Severity:** High. This is an infrastructure regression that directly impacts the reliability of our QA process and our confidence in the app's stability.
