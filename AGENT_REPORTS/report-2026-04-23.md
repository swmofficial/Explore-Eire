## UX Report — 2026-04-23

### Summary
The Playwright test suite is failing to even load the application, preventing any UX validation. The core issue is the Playwright configuration's `baseURL` setting, which is either missing or incorrectly configured, leading to an "invalid URL" error when attempting to navigate to the app's root. This regression blocks all subsequent UI/UX checks.

### Findings

1.  **App Loading Failure in Playwright:**
    *   **Description:** The primary `app.spec.js` test, "app loads and shows splash screen", fails immediately on `page.goto('/')` with a "Cannot navigate to invalid URL" error. This indicates that Playwright cannot resolve the relative path `/` to a full URL, typically because `baseURL` is not correctly configured in `playwright.config.js`.
    *   **Likely Cause from Git Diff:** The recent commit *added* `baseURL: 'http://localhost:3000'` to `playwright.config.js`. However, based on the project memory, Vite's default development server runs on `http://localhost:5173`. The "invalid URL" error suggests that either the `baseURL` setting wasn't effective in the test run, or it's pointing to a non-existent server address. Given Vite's default, a port mismatch is highly probable. The test regression stems from this critical misconfiguration, preventing any app interaction tests.
    *   **Severity:** High (Blocks all UI tests, prevents any UX validation).

2.  **Missing or Failed Screenshot Capture:**
    *   **Description:** Due to the `page.goto` failure, the subsequent `await page.screenshot(...)` command for `01-load.png` could not execute successfully.
    *   **Likely Cause from Git Diff:** Directly consequent to the app loading failure (Finding 1).
    *   **Severity:** Low (Diagnostic tool is broken, but it's a symptom, not the root cause).
