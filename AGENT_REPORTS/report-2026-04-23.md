## UX Report — 2026-04-23

### Summary
The recent Playwright test run has identified a critical regression where the main bottom navigation bar is not visible after the splash screen transitions to the app. Additionally, multiple console errors related to MapLibre GL JS indicate that CSS variables are not being correctly interpreted for map layer paint properties, potentially leading to visual inconsistencies on the map. These issues are surfacing despite the latest commits being infrastructure-related, suggesting either pre-existing bugs now caught by new tests or subtle environmental changes impacting the build/runtime.

### Findings

1.  **Bottom Navigation Bar Not Visible**
    *   **Description:** The Playwright test `"bottom nav is visible"` failed, indicating the primary `nav` element could not be found or was not visible on the page after the splash screen. This prevents users from interacting with the core navigation (Settings, Dashboard, Map, Learn, Profile).
    *   **Likely Cause from Git Diff:** The recent `git diff` shows no changes to application code (`src/` directory), only additions to Playwright tests and GitHub workflows. This suggests the test is exposing a pre-existing bug or a regression caused by factors outside the immediate code changes, such as a dependency update not captured, or a build/environment issue. The test itself was newly added in `app.spec.js`, likely to cover this critical UI component.
    *   **Severity:** High (Critical regression, breaks core app navigation).

2.  **MapLibre GL JS Color Variable Errors**
    *   **Description:** All map-related tests (e.g., `"splash screen transitions to app"`, `"map container is present"`) reported console errors like `[Map] error: layers.session-trail-line.paint.line-color: color expected, "var(--color-accent)" found`. This indicates MapLibre is not resolving CSS custom properties (variables) when defining layer paint styles. This could lead to incorrect or default rendering of map elements like trails, waypoints, and mineral markers.
    *   **Likely Cause from Git Diff:** Similar to the navigation issue, no direct application code changes. This is almost certainly a pre-existing bug related to MapLibre's integration with the app's theming system. MapLibre's style specification typically expects explicit color values (e.g., hex, rgb, named colors), not CSS variables, unless there's an explicit parsing or runtime resolution mechanism in place.
    *   **Severity:** Medium (Visual degradation and console clutter, impacts core map functionality).
