## UX Report — 2026-04-23

### Summary
The recent UX audit, triggered by infrastructure improvements, highlights potential regressions following a crucial commit (`360a79e`) that aimed to fix the `BottomNav` element and standardize MapLibre paint object styling. Despite all `app.spec.js` tests passing, an analysis of the git diff and project memory suggests the possibility of visual inconsistencies and minor interaction bugs within the `BottomNav` and the MapLibre map rendering, particularly concerning mobile responsiveness and layer styling.

### Findings

1.  **BottomNav MapPin Tab Misalignment/Z-Index:** The central "MapPin" tab in the `BottomNav` may appear misaligned, off-center, or experience incorrect z-index issues after the recent fix. This could lead to it visually overlapping other elements or not maintaining its intended "centre-raised" position, especially on different mobile screen sizes.
    *   **Likely Cause:** Commit `360a79e` (`[impl] fix BottomNav nav element`). Refactoring of the `BottomNav`'s layout, potentially affecting specific CSS properties like `position`, `flex` alignment, `transform`, or `z-index` for the unique MapPin button.
    *   **Severity:** Medium (visual regression, impacts primary navigation and user perception)

2.  **BottomNav Haptic Feedback Regression:** The haptic feedback mechanism, expected on `BottomNav` taps as per `CLAUDE.md`, might be non-functional or inconsistent. This would degrade the tactile user experience.
    *   **Likely Cause:** Commit `360a79e` (`[impl] fix BottomNav nav element`). Changes to the `BottomNav` component's event listeners or interaction logic might have inadvertently disconnected the haptic feedback.
    *   **Severity:** Low (minor interaction bug, affects perceived app polish)

3.  **Map Layer Style Inconsistencies:** The replacement of CSS variables within MapLibre paint objects may have introduced unintended visual changes to various map layers. This could manifest as incorrect colors, opacities, line widths, or icon styles for WMS layers (GSI Geochemistry, Bedrock, Boreholes, Met Éireann) or user-generated map data (waypoints, trails, DataSheet geometries).
    *   **Likely Cause:** Commit `360a79e` (`[impl] replace CSS vars in MapLibre paint objects`). Direct manipulation of MapLibre layer styling could have introduced unforeseen side effects or broken consistency with the original design.
    *   **Severity:** High (critical visual integrity issue, impacts core map functionality and data representation)

4.  **Mobile Viewport Issues with BottomNav Responsiveness:** Despite passing basic visibility tests, the `BottomNav` might not be fully responsive across all mobile viewports. Potential issues include incorrect scaling, problematic padding, or encroaching on safe areas, leading to a suboptimal experience on diverse devices.
    *   **Likely Cause:** Commit `360a79e` (`[impl] fix BottomNav nav element`). The `BottomNav` refactor might not have comprehensively addressed all responsive design considerations for a wide range of mobile device dimensions.
    *   **Severity:** Medium (visual regression, degrades mobile user experience)
