## UX Report — 2026-04-23

### Summary
The Playwright UX test for the home page, recently added, is failing due to an inability to navigate to the base URL. This indicates a critical misconfiguration of the Playwright test environment, preventing the application from being properly launched and accessed by the test runner. Consequently, no actual UX issues within the application could be identified.

### Findings

1.  **Playwright Test Failure: Cannot Navigate to URL**
    *   **Description:** The test `home page loads with map` in `home.spec.js` failed with the error `Error: page.goto: Protocol error (Page.navigate): Cannot navigate to invalid URL`. This occurs when `await page.goto('/');` is executed.
    *   **Likely Cause:** The Playwright test runner is attempting to navigate to a relative path (`/`) without a `baseURL` configured in `playwright.config.js`, and critically, without a `webServer` configured to launch the application. The Playwright results confirm `webServer: null`. This means the application under test was not running or accessible for the test to interact with. The git diff shows `home.spec.js` was just added, suggesting this is a new infrastructure setup issue rather than an application regression.
    *   **Severity:** High (Blocks all automated UX testing, rendering the new test suite ineffective).

### Actionable Prompts for Agents

1.  **[infra] Configure Playwright `webServer` and `baseURL`**
    The Playwright test setup is failing because the application is not being launched for the tests to run against. Update `playwright.config.js` to include a `webServer` configuration that starts the Vite development server before running tests, and set a `baseURL` for tests to navigate to. This will allow `page.goto('/')` to correctly point to the running application.

    **Example `playwright.config.js` addition:**
    ```javascript
    import { defineConfig, devices } from '@playwright/test';

    export default defineConfig({
      // ... existing config ...
      webServer: {
        command: 'npm run dev', // Command to start your Vite dev server
        url: 'http://localhost:5173', // Adjust port if different, Vite's default is 5173
        timeout: 120 * 1000, // Give it time to start
        reuseExistingServer: !process.env.CI, // Reuse server locally, always start in CI
      },
      use: {
        baseURL: 'http://localhost:5173', // This should match the webServer url
        // ... other use options ...
      },
      // ... rest of config ...
    });
    ```
