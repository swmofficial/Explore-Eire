## UX Report — 2026-04-23

### Summary
Playwright tests passed, but a recent commit (`360a79e`) made significant changes to `BottomNav.jsx` and `Map.jsx` that are highly susceptible to introducing visual regressions and interaction issues. Specifically, the replacement of CSS variables in MapLibre paint objects in `Map.jsx` is a critical concern for map layer rendering, and modifications to the `BottomNav` element could lead to z-index conflicts or mobile responsiveness problems.

### Findings

1.  **Map Layer Visual Regressions**
    *   **Description:** Map layers (e.g., WMS layers, gold tiers) might be rendering with incorrect colors, opacities, line widths, or may even be entirely absent on the map. This visual discrepancy is particularly expected for elements previously styled using CSS variables.
    *   **Likely Cause from Git Diff:** Commit `360a79e` - `[impl] fix BottomNav nav element + replace CSS vars in MapLibre paint objects`. The "replace CSS vars in MapLibre paint objects" part in `src/components/Map.jsx` is the direct cause, where CSS variables might have been replaced by hardcoded values or incorrect references, leading to unintended styling.
    *   **Severity:** High (Directly impacts core map functionality and data visualization, which is central to Explore Eire's value proposition).

2.  **Bottom Navigation Bar Overlap/Z-index Issues**
    *   **Description:** The `BottomNav` element may be overlapping other critical UI components (e.g., `DataSheet`, `LayerPanel`, `BasemapPicker`) or itself being overlapped, particularly affecting the visually distinctive `MapPin tab centre-raised` element. This could manifest as elements appearing above the nav bar, or parts of the nav bar being obscured, hindering interaction.
    *   **Likely Cause from Git Diff:** Commit `360a79e` - `[impl] fix BottomNav nav element`. A structural or styling "fix" to the nav element in `src/components/BottomNav.jsx` could have inadvertently altered its z-index or positioning relative to other floating UI elements, overriding the intended `zIndex: 40` specified in project memory.
    *   **Severity:** Medium (Affects overall UI usability and aesthetic integrity).

3.  **Bottom Navigation Bar Mobile Responsiveness Issues**
    *   **Description:** On mobile viewports (various screen sizes and orientations), the `BottomNav` might exhibit layout issues such as truncated text, misaligned icons, or incorrect padding/spacing, especially concerning the `MapPin tab centre-raised` feature.
    *   **Likely Cause from Git Diff:** Commit `360a79e` - `[impl] fix BottomNav nav element`. Changes to the base navigation element in `src/components/BottomNav.jsx` without thorough responsive testing could break existing mobile layouts.
    *   **Severity:** Medium (Critical for a mobile-first application; directly impacts user experience on handheld devices).
