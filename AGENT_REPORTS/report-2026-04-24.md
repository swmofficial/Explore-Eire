## UX Report — 2026-04-24

### Summary
Playwright tests for the `DataSheet` component are failing due to an ambiguous selector for the "Map" navigation button, leading to a strict mode violation. This regression was introduced by recent changes to improve test coverage. The issue highlights a potential UX and accessibility problem with overlapping accessible names for interactive elements within the application.

### Findings

1.  **Playwright Test Failure - Ambiguous "Map" Navigation Button**: The `sheets.spec.js` test suite is entirely failing at the navigation step, specifically `await page.getByRole('button', { name: 'Map' }).click();`. Playwright reports a strict mode violation, identifying four elements matching the `name: 'Map'` criteria. These include the intended "Map" navigation button (which has `aria-label="Map"`) and three other functional buttons: "My Waypoints No waypoints yet", "My Sessions No sessions yet", and "Offline Maps No offline areas". This prevents tests from reaching the `DataSheet` component.
    *   **Likely Cause from Git Diff**: Commit `7bee4bb` ("[impl] Playwright coverage gap — tasks 007/008/009") introduced the explicit navigation to the "Map" tab in `sheets.spec.js` (and similar navigation patterns in `settings.spec.js`, `dashboard.spec.js`, `learn.spec.js`). This change exposed a pre-existing ambiguity in the UI's accessible naming scheme, where multiple buttons were inadvertently matched by the broad `name: 'Map'` selector.
    *   **Severity**: **High**. This issue blocks critical UX tests for the `DataSheet` component and indicates a potential accessibility concern where users of assistive technologies might encounter ambiguity when trying to navigate or interact with buttons.
