1.  **Bottom Navigation Overlap/Incorrect Positioning on Mobile:**

    *   **Description:** Visual elements within MapLibre-rendered layers, such as WMS layers (GSI Geochemistry, Bedrock, Boreholes) or custom prospecting data, display with incorrect colors, opacities, line widths, or are otherwise visually inconsistent with their intended design. Some layers might even be partially or fully invisible.
    *   **Likely Cause:** Commit `360a79e` ("[impl] ...replace CSS vars in MapLibre paint objects") involved replacing CSS variables within MapLibre paint properties. Errors in this replacement process could lead to incorrect style application.
    *   **Severity:** Medium (affects the primary data visualization and core value proposition of the map).

3.  **Missing or Inconsistent Haptic Feedback on BottomNav:**
    *   **Description:** The specified "Haptic feedback on tap" for `BottomNav` items is either entirely absent or inconsistent across taps, particularly for the central MapPin button.
    *   **Likely Cause:** Commit `360a79e` ("[impl] fix BottomNav nav element"). Structural or event handler changes to the `BottomNav` may have inadvertently broken the integration of haptic feedback.
    *   **Severity:** Low (degrades premium user experience, but not critical functionality).
